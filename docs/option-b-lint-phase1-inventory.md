## The ban rule implemented (conservative reading)
"No bare access to a contract-scoped repository. Contract-scoped reads / by-id loads
must go through a scoped repository (the chokepoint)." Detected patterns (OUTSIDE the
chokepoint): `@InjectRepository(Entity)`, `getRepository(Entity)`, `<repo>.<dataMethod>()`
on a `Repository<Entity>`, `<manager>.<dataMethod>(Entity, …)` — where Entity is contract-scoped.
Implementation: a real custom **ESLint rule** (syntactic, no type-checker), so Phase 2 can flip `warn`→`error` cleanly.
